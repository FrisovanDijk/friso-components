Friso's Component API Docs

Why a component API?

I enjoy using new technologies with every project, but that renders a lot of front-end work useless because it's heavily embedded in different templating languages and CMSes. You end up refactoring and rewriting.

A Component API decouples small, reusable components from their projects. Small, standardised components that are the building blocks of something bigger.

The idea is very simple: you make a request with the component name and JSON, you will receive its semantic component. Here are my reasons for building this API:

  1. Agnostic. The implementation before you is lightweight and doesn't require a JS framework, but nothing stops you from using one.
  2. Reduced complexity. Managing a single API makes changes to components easy.
  3. Reuse. All components come from a single source. It's easy to grab your best semantic, accessible HTML. Class names are the same across projects.
  4. Discoverable. Browse the docs with examples of components to see what would work for you.
  5. Readable. Your project code become extremely readable by replacing more complex HTML with clear JSON.
  6. Simple. It takes 5 minutes to set up and improves readability.
  7. Portable. The API uses Mustache as its sole dependency, which has an implementation in most popular programming languages. This and its small implementation size (1 file) make it extremely portable between languages if so desired.

Take a look around. The project is open source and available on Github, or get inspired and make your own.

Installing Friso's Component API

Get it from Github and run composer install. You will see 5 items:

  • css: holds the default templates. Host them with the API to get consistent styling across all your apps.
  • dev: the documentation you're reading right now.
  • templates: the mustache templates used on the server
  • vendor: required packages.
  • index.php: this is the API handler.

Configuration

If you want to use the Javascript implementation used here, then you need to grab components.js from dev.

When you create a new template, upload it to the templates folder or create a new listener. Register your template by adding it to static $components at the top of index.php

Using the Component API

Using it in a project is extremely simple. This documentation site uses a very light Javascript implementation (although it doesn't support IE).

If you use it, make sure you configure const url for your own configuration.

To render a component with it, simply use the Javascript code below:

parse_componentHTML(identifier, {
"name": component name,
"vars": {
    component variables
}
});

Ok, so what can I input? The identifier is your in-document identifier and uses document.querySelector(). The second argument is the JSON to generate the component. The name is the type of component you generate, and vars is the food for your component.

About these docs

The docs are my personal templates with my branding applied. They specify my template variables and show the json behind the components on the page.

Feel free to reuse or change everything as preferred.

Switching the branding on the default template is easy to do. Open _variables.scss and change the relevant colours. Compile the stylesheet and you're good to go (or check out Koala if you don't know how)
